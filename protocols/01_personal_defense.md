**"Vibe Coding"** is the practice of accepting AI-generated code because it *looks* right, without verifying if it *is* right. It is the primary source of the Subprime Code Crisis. 

Before we change the industry, we must discipline our own workflow. We are the architects of logic, not consumers of syntax.

---

### 1. The Containment Rule

Until AI demonstrates true architectural reasoning and context awareness, restrict autonomous code generation to "Safe Zones." Do not let the AI dictate the architecture.

#### ðŸŸ¢ Green Zone (AI Allowed)
*Use AI freely for these tasks to accelerate throughput:*
*   **Disposable Scripts:** One-off data migration or parsing scripts.
*   **Data Transformation:** Converting JSON to SQL, mapping types.
*   **Boilerplate:** Generating standard class structures, getters/setters (if not using Lombok/Records).
*   **Unit Test Scaffolding:** Generating the *structure* of tests (but review the assertions manually).
*   **Documentation Drafts:** Initial Javadoc/Docstring generation.

#### ðŸ”´ Red Zone (Human Only)
*Do not accept AI output without forensic audit:*
*   **Core Business Logic:** The algorithms that define the company's value proposition.
*   **Security Controls:** AuthZ/AuthN logic, encryption handling, input validation.
*   **State Management:** Complex React hooks, database transaction boundaries, concurrency controls.
*   **Architectural Decisions:** Choosing patterns, defining API contracts, system design.

### 3. The Commitment

To maintain professional integrity, adopt this stance:

> **"Refuse to normalize 'coding by probability'. If I didn't read it, I didn't write it. If I don't understand it, I won't commit it."**

You are the signatory of the code. The `git blame` points to you, not the LLM. Act accordingly.
